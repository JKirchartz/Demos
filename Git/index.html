Look Smarter With Git

A short guide to git tricks and worthwhile aliases

See what's up

$> git log # list previous commits
$> git log -p path/to/file.name # show commits to a single file
$> git log --name-status HEAD^..HEAD
$> git log --graph --format=oneline --abbrev-commit
$> git show #Show details of last commit, including diff

Show shorter details of last commit

Save Alias: 

git --global alias.last = 'log --name-status HEAD^..HEAD'

show prettier list of previous commits

Save Alias: 

git --global alias.lo = 'log --graph --format=oneline --abbrev-commit'

Changing History

Warning: Altering git's history is potentially dangerous when you've already published the history

Use 

git push --force
only if you're absolutely certain you're not causing headaches for other developers

Forgot to add a file?

Add to last commit

$>git add path/to/file.ext
$>git commit --amend

Note: if you've already pushed your previous commit to remote, you may cause headaches for other developers missing your forgotten files

Add all changes to last commit, Keep commit message

$>git commit --all --ammend --no-edit

`--all` adds all changed files

`--no-edit` skips editing the commit message

Save Alias: 

git --global alias.caa = 'commit --all --ammend --no-edit'

Add the wrong file?

Unstage Changes

$>git reset HEAD --

Save Alias: 

git --global alias.unstage = 'reset HEAD --'

Undo Last commit

$>git reset HEAD~1

Save Alias: 

git --global alias.undo = 'reset HEAD~1'

Cover your Tracks

$>git rebase -i HEAD~5

Notes: Do not rebase commits you've already pushed to remote, you will give everyone headaches

`-i` enables interactive mode, `HEAD~5` selects the last 5 commits for revision.

In interactive mode you can combine commits, edit commit messages, you can even re-order commits by rearranging the lines in the commit message

# Commands:
#  p, pick = use commit
#  r, reword = use commit, but edit the commit message
#  e, edit = use commit, but stop for amending
#  s, squash = use commit, but meld into previous commit
#  f, fixup = like "squash", but discard this commit's log message
#  x, exec = run command (the rest of the line) using shell
#
# These lines can be re-ordered; they are executed from top to bottom.
#
# If you remove a line here THAT COMMIT WILL BE LOST.
#
# However, if you remove everything, the rebase will be aborted.
#
# Note that empty commits are commented out


$>git pull --rebase

Use rebase instead of merge when you pull a commit

Replay your commits on top of the remote branch

Avoid pointless merge messages

$>git pull --rebase=preserve

Use rebase instead of merge when you pull a commit

Replay your commits on top of the remote branch

Keep pointless merge messages
